Local-first workspace helper

Start using devdrop.

devdrop keeps a developer workspace legible across machines and agents: the tree, sync state, ignored generated files, encrypted secrets, repo status, and reviewable agent overlays.

Quickstart Daily use Sync Secrets Agents Health

Quickstart

Build the binary, create a workspace, enroll the current machine, then scan the workspace into the local SQLite index.

cargo build --release
./target/release/devdrop workspace init ~/code
cd ~/code
devdrop login
devdrop device enroll "MacBook Pro"
devdrop sync
devdrop status
~/code
  work/
    api/
    web/
  personal/
    blog/
  experiments/
    prototype/

The model

1. WorkspaceChoose one root, usually ~/code. devdrop stores metadata in .devdrop/.
2. Scandevdrop sync records files, repo status, blobs, tombstones, and ignore rules.
3. HydrateRemote-only content can be fetched later with devdrop hydrate.
4. ReviewAgent work lands in overlays before it touches the real repo.

Daily use

See the workspace

devdrop status shows local, ignored, remote-only, locked secret, conflict, and repo counts.

List with state

devdrop ls ~/code/work prints entries as local, ignored, metadata-only, remote-only, or secret-locked.

Keep important files

devdrop pin path marks a path as important. Use unpin when it no longer needs priority.

Useful commands

devdrop ignored [path]Show files excluded by built-in rules or .devsyncignore.
devdrop repo-status [path]List Git branches, dirty state, ahead/behind counts, and stale-base warnings.
devdrop repo update [path]Run a safe git fetch --prune and fast-forward merge for the repo upstream.
devdrop history pathShow known versions for a file.
devdrop recover path [--hash hash]Restore the latest known file version or a specific content hash.

Sync between machines

The current product uses a file-backed remote. Put it on a shared disk, network mount, or synced folder while the cloud service is still out of scope.

# On the shared location
devdrop remote init /mnt/devdrop-remote

# On machine A
devdrop sync ~/code --remote /mnt/devdrop-remote

# On machine B
devdrop workspace mount ~/code
devdrop sync ~/code --remote /mnt/devdrop-remote --pull
devdrop ls ~/code

Generated folders such as node_modules, target, .venv, caches, logs, and secret files are classified by default so they do not sync as ordinary source.

Secrets

Secrets are encrypted locally with OpenSSL and tracked as locked metadata until explicitly unlocked or requested for a process.

export DEVDROP_SECRET_KEY="choose-a-local-passphrase"
devdrop secret add ~/code/work/api/.env --scope dev
devdrop secret lock ~/code/work/api/.env --scope dev
devdrop secret request ~/code/work/api/.env --scope dev
devdrop run --repo ~/code/work/api --secret-scope dev -- cargo test

Agent overlays

Agents work in an overlay copy. Review the diff, submit it, then accept or reject it. Accepting enforces the stored write scope.

devdrop agent create --repo ~/code/work/api --write-scope 'src/**' --secret-scope test
devdrop agent status
devdrop overlay diff <agent-id>
devdrop overlay submit <agent-id>
devdrop agent accept <agent-id>

For machine-readable agent calls, use devdrop status --json, devdrop repo-status --json, and devdrop secret request.

Health checks

Run doctor when setup feels wrong, a remote is missing, auth/device state is unclear, or a tool dependency is not installed.

devdrop doctor ~/code
devdrop conflicts ~/code
devdrop conflicts resolve ~/code/work/api/src/config.ts --use base
